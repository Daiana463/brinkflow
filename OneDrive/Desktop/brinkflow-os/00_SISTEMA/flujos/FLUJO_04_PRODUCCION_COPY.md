# FLUJO 04 — PRODUCCIÓN DE COPY DE ANUNCIO

## Propósito

Producir copy ejecutable para anuncios (Reels, imagen, carrusel) con criterio senior, listo para que Vero apruebe y se publique.

**Cuándo invocar:** cada vez que se necesitan creativos nuevos. Frecuencia típica: semanal o quincenal según plan.

**Output esperado:** 3 variantes de copy por ángulo, con hook + cuerpo + CTA, cada una con justificación de qué principio activa.

---

## Cómo invocar este flujo

> "Flujo 04 — 3 variantes de copy para [cliente]. Ángulo: [cual]. Formato: [Reel / imagen / carrusel]. Adjunto BRIEF, perfiles, Big Idea y MEMORY."

---

## Lo que Claude debe hacer (instrucciones internas)

### Antes de escribir, razoná internamente

1. **Confirmá el nivel de conciencia (Schwartz) del perfil objetivo.** Define el tipo de hook:
   - Nivel 2-3 → hook de problema reconocido o verdad incómoda
   - Nivel 3-4 → hook de solución específica o autoridad
   - Nivel 4-5 → hook de fricción mínima, CTA directo

2. **Identificá la objeción dominante real** (del MEMORY si hay datos offline, del BRIEF si no). Aplicá reencuadre — no defiendas la posición de la marca, mostrá el costo de no actuar.

3. **Construí cada variante desde un principio distinto**. No 3 versiones del mismo. Tres archetypes:
   - **Tensión reconocida**: el cliente se ve reflejado en el problema antes de ofrecerle la solución.
   - **Verdad incómoda**: comunicás algo que el cliente ya sabe pero nadie se lo dijo así.
   - **Especificidad radical**: destruís lo genérico con datos concretos.
   - O variantes: autoridad sin arrogancia, transformación respetuosa, fricción eliminada, contraste.

4. **Aplicá principios de copywriting senior**:
   - **Slippery slope (Sugarman)**: cada línea hace que la siguiente sea irresistible.
   - **Specificity over generality (Halbert)**: "4 sesiones" gana a "varias sesiones".
   - **One idea, one piece (Caples)**: una sola idea por anuncio. Si tiene dos, no tiene ninguna.
   - **Hook en los primeros 3 segundos** (video) o **primera línea** (estática).

5. **Aplicá test de intercambiabilidad ANTES de devolver el output.** Si el copy podría ser firmado por la competencia, lo reformulás.

6. **Pasá por filtro de COMPLIANCE_RULES**. Si activa palabras en cuarentena o cualquier veto, reformulás antes de devolver. No "produzcas y filtres después" — filtrá durante.

7. **Validá coherencia con el tono del cliente** (BRIEF → tono de comunicación) y con BrinkFlow_DNA.

## Frameworks senior que activás

- **Hook engineering** (David Ogilvy, Joe Sugarman): el hook es el 80% del trabajo.
- **Specificity (Gary Halbert)**: específico siempre vence a genérico.
- **Slippery slope (Sugarman)**: cada línea es un trampolín a la siguiente.
- **Niveles de conciencia (Schwartz)**: el mensaje correcto depende del nivel.
- **Test de intercambiabilidad**: si la competencia podría firmarlo, no es nuestro.
- **Reencuadre psicológico** (Cialdini, Moesta): la objeción se disuelve cambiando el marco, no defendiéndolo.

---

## Qué hace que este copy sea senior y no genérico

- Cada variante usa un **principio psicológico distinto**, no es matiz del mismo.
- Cada variante **justifica explícitamente** qué activa y por qué para este perfil.
- Pasó por **test de intercambiabilidad** antes de salir.
- Tiene **una sola idea** por variante, no tres.
- El hook **interrumpe**, no describe.
- El CTA es **único y claro**, no es "conocé más / contactanos / seguinos".
- **Todo pasó por compliance** durante la producción, no después.
